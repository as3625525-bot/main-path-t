What actually mattered in that conversation

Strip away the back-and-forth and three decisions are doing all the work. Everything else in this page is just the execution detail underneath them.

1

One path, not six

AI engineering, GitHub, certificates, internships, trading, English, US logistics — all named in the same breath. The fix wasn't a better list. It was picking one main path (Python → SQL → ML → cloud) and demoting everything else to secondary.

2

Proof beats certificates

Employers were described as caring about practical ability and shipped projects first, credentials second. A resume screener or sentiment analyzer on GitHub outweighs a stack of course completions.

3

Trading stays in its lane

A 60% demo win rate was flagged directly as not dependable — demo performance reliably overstates real-money performance. Trading is a side skill. The income plan is AI/software/data, full stop.

the honest read

You're not behind. The plan called the starting position good — reflection, ambition, and early GitHub activity already in place. What was missing wasn't motivation. It was sequence.

phase 0 · months 1–6

Foundations

The goal here isn't impressive. It's fluency — being able to build without getting stuck every five minutes.

Build

  • Python fundamentals, properly — not just syntax tourism
  • Git, GitHub, basic Linux
  • SQL queries and database basics
  • 2 small projects, shipped in public

Maintain in parallel

  • English — reading, speaking, writing, every day
  • Trading — demo only, capped time, side interest
exit signal

You stop Googling the same Python syntax twice. You can explain what your project does to someone who isn't you.

phase 1 · months 7–12

Applied ML

This is where tutorials end and actual modeling starts: cleaning data, training, evaluating, and learning where models break.

Core concepts

  • Data preprocessing and feature engineering
  • Model training and evaluation
  • Overfitting — recognizing and fixing it
  • pandas, NumPy, scikit-learn

Ship 2–3 of these

  • Spam classifier
  • Price prediction model
  • Recommendation system
  • Resume screener
  • Sentiment analyzer
  • Simple chatbot
exit signal

Your GitHub shows you can use ML, not just narrate a course you watched.

phase 2 · months 13–18

Job-oriented skills

The shift from "learner" to "candidate." This phase exists to make your portfolio look like an internship application, not a study log.

Add

  • Deep learning basics
  • APIs and how to consume/build them
  • Cloud fundamentals (pick one provider)
  • Docker and basic deployment
  • 1 fully deployed project
  • One certificate — only if it reinforces what you're learning, not for its own sake
exit signal

You look like an internship candidate. Not a beginner with a longer GitHub history.

phase 3 · months 19–24

Career mode

Skill-building slows down on purpose. Output shifts to applications, polish, and getting in front of people.

Do

  • Apply for internships and part-time technical roles
  • Build a portfolio website
  • Tighten resume and LinkedIn
  • Practice interviews — out loud, repeatedly
  • Ship one new project every 1–2 months, no exceptions
exit signal

Ready for junior work, internship interviews, or part-time technical roles — and ready to walk into a US bachelor's program with proof in hand, not just a plan.

the money question

What this is actually worth

Real numbers discussed in the conversation, kept honest about the gap between "possible" and "likely on day one."

starting out · no US experience $15–$30/hr Part-time / student tech work — IT support, QA, web or Python assistant roles
ai / ml internship $20–$50/hr Most land mid-$20s to $30s. $50/hr needs a portfolio that proves independence, not just theory.
entry-level full-time $70k–$120k/yr Once qualified for a full role — the payoff after the portfolio, not before it

What a $50/hr internship actually expects

Strong Python + problem-solving, not beginner level
SQL and real data handling
3–5 solid GitHub projects you can explain clearly
Enough independence to finish tasks with limited hand-holding
Sometimes: cloud basics or model deployment

If you're still building the foundation, this rate isn't the first step — it's roughly where phase 2–3 lands you, not phase 0.

the side door

Trading: keep it, but keep it small

A 60% win rate on demo was the number on the table. Demo performance consistently overstates real-money performance — the conditions that produce a 60% demo number rarely survive contact with real capital, slippage, and emotion.

Trading isn't being cut from the picture. It stays as a discipline-and-analysis skill. What it can't be is the backup plan, and it especially can't be the plan while an F4-based US move and family responsibilities are still ahead.

main income plan → AI / software / data skills

What arriving in the US actually looks like

The bachelor's plan gets dramatically stronger if these are already true on the day you land — not started after.

✓Solid Python and SQL
✓Several real GitHub projects
✓Working ML knowledge
✓Good spoken and written English
✓One or two certificates — supporting, not central

why this order, specifically

A bachelor's degree plus projects and internship experience was framed as meaningfully stronger than a degree alone. The degree is the credential; the portfolio is the proof the credential is real.

closing log entry

5 rules, no exceptions

  1. 01Study hard, on the one path — not six half-paths
  2. 02Build something every month, no gaps
  3. 03Post it on GitHub — unfinished is fine, invisible is not
  4. 04English daily — it compounds as quietly as the code does
  5. 05Keep trading controlled — a skill, never the plan
